Git Submodule 에러와 구글 애널리틱스 연동 트러블슈팅기

May 26, 2026

## 1. 문제 상황 (Background)

Hugo를 활용해 GitHub Pages 블로그를 구축하고 구글 애널리틱스를 연동하는 과정에서, Git Submodule 명령어가 거부되고 Repository not found 및 A git directory for 'public' is found locally 등의 연쇄적인 에러를 마주했습니다.

## 2. 원인 분석 (Root Cause)

* **경로 및 동기화 에러:** Windows의 OneDrive 및 한글 경로가 포함된 바탕화면은 CLI 도구와 충돌을 유발했습니다.

* **식별자 불일치:** GitHub 저장소 생성 시 xoxo787.github.io 규격을 지키지 않고 github.io로 잘못 생성하여 터미널이 대상을 찾지 못했습니다.

* **이력(Commit) 충돌:** 구글 연동을 위해 GitHub 웹상에서 먼저 생성한 README.md 파일로 인해 원격과 로컬의 히스토리가 어긋나 git push가 거부되었습니다.

## 3. 해결 방법 (Resolution)

문제를 해결하기 위해 작업 환경을 영문 전용 경로(C:\\workspace)로 격리한 후 다음 명령어로 내부 찌꺼기를 청소하고 강제 동기화를 진행했습니다.


\# 1. 잘못 꼬인 서브모듈 설정 및 캐시 강제 청소

rm -r -force public

rm -r -force .git/modules/public

git rm --cached public



\# 2. 정확한 URL 주소로 서브모듈 재연결

git submodule add -b main \[https://github.com/xoxo787/xoxo787.github.io.git](https://github.com/xoxo787/xoxo787.github.io.git) public



\# 3. 테마 결합 및 로컬 클린 빌드

git submodule add \[https://github.com/theNewDynamic/gohugo-theme-ananke.git](https://github.com/theNewDynamic/gohugo-theme-ananke.git) themes/ananke

hugo



\# 4. 원격 이력 충돌 무력화 및 강제 배포

cd public

git push origin main --force
