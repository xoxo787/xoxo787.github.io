## 1. 아키텍처 확장 배경 (Problem & Objectives)

단일 타임라인(posts)으로 운영되던 기존 정적 블로그 구조에서 벗어나, 전문적인 **‘기술 기록(Tech)’**의 영역과 인간적인 **‘삶의 기록(Life)’**이라는 이종(異種)의 데이터 레이어를 논리적으로 완전 격리하고, 본문 내부에 깨짐 없는 정적 이미지 자산(Asset)을 무손실 매핑할 수 있는 독립형 정보 아키텍처(Information Architecture) 구축을 목표로 합니다.

## 2. 디렉터리 기반 라우팅 설계 (Directory-based Routing)

Hugo 프레임워크의 엄격한 데이터 스키마 규칙을 준수하여, content 부모 보관함 하위에 독립된 서브 디렉터리를 생성함으로써 물리적·논리적 라우팅 경로를 분리했습니다.

* content/tech/: 기술 문서 마크다운 원고 보관 구역 (.../tech/ URL로 맵핑)

* content/life/: 삶의 에세이 마크다운 원고 보관 구역 (.../life/ URL로 맵핑)

## 3. 글로벌 설정 파일 제어 및 UI 매핑 (hugo.toml)

사용자가 웹 인터페이스 상에서 두 카테고리를 자유롭게 오갈 수 있도록 전역 환경 설정 파일 최상위 메뉴 객체(menus.main)에 내비게이션 바 변수를 선언하고 가중치(weight)를 할당하여 UI 접근성을 최적화했습니다.


\[\[menus.main]]

&#x20;   name = "기술 기록"

&#x20;   url = "/tech/"

&#x20;   weight = 10



\[\[menus.main]]

&#x20;   name = "삶의 기록"

&#x20;   url = "/life/"

&#x20;   weight = 20
