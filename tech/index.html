## 1. 아키텍처 확장 배경 (Problem & Objectives)

단일 타임라인(posts)으로 운영되던 기존 정적 블로그 구조에서 벗어나, 전문적인 **‘기술 기록(Tech)’**의 영역과 인간적인 **‘삶의 기록(Life)’**이라는 이종(異種)의 데이터 레이어를 논리적으로 완전 격리하고, 본문 내부에 깨짐 없는 정적 이미지 자산(Asset)을 무손실 매핑할 수 있는 독립형 정보 아키텍처(Information Architecture) 구축을 목표로 합니다.

## 2. 디렉터리 기반 라우팅 설계 (Directory-based Routing)

Hugo 프레임워크의 엄격한 데이터 스키마 규칙을 준수하여, content 부모 보관함 하위에 독립된 서브 디렉터리를 생성함으로써 물리적·논리적 라우팅 경로를 분리했습니다.

Git Submodule 에러와 구글 애널리틱스 연동 트러블슈팅기

## 1. 문제 상황 (Background)

Hugo를 활용해 GitHub Pages 블로그를 구축하고 구글 애널리틱스를 연동하는 과정에서, Git Submodule 명령어가 거부되고 Repository not found 및 A git directory for 'public' is found locally 등의 연쇄적인 에러를 마주했습니다.

## 2. 원인 분석 (Root Cause)

* **경로 및 동기화 에러:** Windows의 OneDrive 및 한글 경로가 포함된 바탕화면은 CLI 도구와 충돌을 유발했습니다.

* **식별자 불일치:** GitHub 저장소 생성 시 xoxo787.github.io 규격을 지키지 않고 github.io로 잘못 생성하여 터미널이 대상을 찾지 못했습니다.
