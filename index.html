깃허브 블로그에 나만의 비밀 일기장 만드는 법 (카테고리 숨기기)

## 1. 남들에게 보여주기 부끄러운 솔직한 기록들

기술 블로그를 운영하다 보면, 가끔은 정돈된 개발 지식 말고 날것 그대로의 감정이나 찌질하고 솔직한 고민들을 적어두고 싶을 때가 있습니다. 하지만 전 세계 누구나 볼 수 있는 인터넷 공간에 내 깊은 속마음을 모두 공개하기는 너무 창피하고 부담스럽죠. 그래서 기술 블로그 안에 나 혼자만 볼 수 있는 ‘비밀 일기장’ 공간을 숨겨놓는 방법을 적용해 보았습니다.

## 2. 마법의 주문: draft = true

Hugo 블로그에서는 글을 인터넷에 올릴지 말지를 아주 간단한 단어 하나로 결정할 수 있습니다. 바로 글 맨 위에 있는 설정창의 draft 속성입니다.

블로그 카테고리(tech/life) 나누고 이미지 넣는 방법

1. 카테고리를 나눈 이유

블로그에 글을 쓰다 보니, 개발을 공부한 ‘기술 기록’과 평범한 일상을 적은 ‘삶의 기록’이 한곳에 섞여 있는 게 보기 불편했습니다. 이 두 가지를 깔끔하게 분리하고, 글 중간에 사진도 예쁘게 넣고 싶어서 블로그 구조를 바꿔보았습니다.

2. 폴더를 나누면 카테고리가 됩니다

Hugo 블로그는 폴더를 나누는 것만으로도 인터넷 주소가 분리됩니다.

  • content/tech/: 기술 관련 글을 모아두는 곳
  • content/life/: 일상 에세이를 모아두는 곳

3. 상단 메뉴바에 버튼 만들기

카테고리를 나눴으니 사람들이 클릭할 수 있게 메뉴 버튼을 만들어야 합니다. 블로그 전체 설정을 담당하는 hugo.toml 파일 맨 밑에 아래 코드를 추가했습니다.

Git Submodule 에러와 구글 애널리틱스 연동 해결기

1. 멘붕의 시작

블로그에 구글 애널리틱스를 달아서 방문자를 확인해 보려다가 에러 파티를 만났습니다. 터미널 창에 Repository not found 같은 무서운 빨간 글씨들이 쏟아지고 배포가 멈췄을 땐 정말 당황했죠. 하지만 천천히 원인을 찾고 해결해 나간 과정을 기록해 봅니다.

2. 도대체 왜 그랬을까?

찬찬히 살펴보니 제가 놓친 사소한 실수 세 가지가 겹쳐 있었습니다.

  • 폴더 경로 문제: 윈도우 바탕화면(한글 경로와 클라우드 연동)에서 작업하다 보니 프로그램들이 파일 위치를 잘 못 찾고 엉켰습니다.
  • 주소 오타: 깃허브 저장소를 만들 때 내아이디.github.io라고 정확히 써야 하는데, 실수로 github.io라고만 적는 바람에 터미널이 길을 잃었습니다.
  • 꼬여버린 순서: 깃허브 웹사이트에서 README.md 파일을 먼저 만들어버린 탓에, 내 컴퓨터와 인터넷 서버의 파일 순서가 어긋나서 밀어올리기(push)가 완전히 막혀버렸습니다.

3. 이렇게 해결했습니다!

작업 폴더를 윈도우의 가장 깔끔한 영문 경로(C:\workspace)로 옮기고, 꼬인 파일들을 하나씩 정리했습니다.
